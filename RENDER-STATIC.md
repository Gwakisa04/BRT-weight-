# Deploy frontend on Render (Static Site)

## Render settings

| Field | Value |
|--------|--------|
| **Service type** | Static Site |
| **Build Command** | `npm install && npm run build` |
| **Publish Directory** | `out` |

### What is `public/` vs `out/`?

- **`public/`** — Source files (bus.webp, icons). Copied into the build automatically. **Not** the publish folder.
- **`out/`** — Generated by `npm run build` after `output: 'export'`. **This** is the Publish Directory.

## Environment variables

Set in Render **before** build (values are baked into the static JS):

```env
NEXT_PUBLIC_API_URL=https://brt-weight-backend.onrender.com/api
NEXT_PUBLIC_SOCKET_URL=https://brt-weight-backend.onrender.com
NEXT_PUBLIC_GPS_POLL_INTERVAL_MS=5000
```

`BACKEND_URL` is **not used** on static hosting. Login talks to the API directly from the browser.

## Backend CORS

On **brt-weight-backend** (Render → Environment), include your static site URL:

```env
CORS_ORIGIN=https://brtweight.onrender.com,http://localhost:3000
```

Or only:

```env
FRONTEND_URL=https://brtweight.onrender.com
```

Redeploy the **backend** after changing env vars.

## After deploy

1. Open `https://YOUR-FRONTEND.onrender.com/login/`
2. Sign in with a user from the backend database
3. If login fails with a network/CORS error, fix `CORS_ORIGIN` on the backend
